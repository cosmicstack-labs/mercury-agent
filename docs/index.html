Soul-Driven Identity

Mercury isn't a blank slate. Its personality is defined by markdown soul files — soul.md, persona.md, taste.md, heartbeat.md — that you control. No corporate wrapper. Your agent, your rules.

⚡

Token-Efficient

Daily token budgets with enforcement. Only soul + persona injected per request (~400 tokens). Skill summaries loaded progressively. Auto-concise mode when budget exceeds 70%.

♾

Always On

Background daemon by default. Cron scheduling, delayed tasks, and a heartbeat system that monitors budget and upcoming tasks. Auto-starts on boot, auto-restarts on crash. Use --foreground for attached mode.

Multi-Channel

CLI with readline and arrow-key command menus. Telegram with typing indicators, HTML formatting, file uploads, and private 1:1 access approval. More channels coming: Signal, Discord, Slack, WhatsApp.

Skill System

Install community skills with a single command. Skills auto-load into context, get elevated permissions, and can be scheduled as recurring tasks. Based on the Agent Skills specification.

Folder-level read/write scoping. Command blocklist (sudo, rm -rf, etc. — never executed). Pending approval flow. Ask Me or Allow All mode per session.

🧠 Second Brain

Most AI agents forget everything when you close the chat. Mercury remembers — automatically, privately, and with surgical precision.

Learns Automatically

After each conversation, Mercury extracts facts about you — your preferences, goals, projects, habits, relationships, and decisions. No manual entry needed.

Recalls What Matters

Only memories relevant to the current conversation are injected — up to 5 facts within a 900-character budget. Your context window stays clean.

Resolves Conflicts

Preferences change. When Mercury detects a contradiction (you liked X, now you prefer Y), the higher-confidence memory wins. No stale data.

10 Memory Types

Identity, preference, goal, project, habit, decision, constraint, relationship, episode, and reflection — each scored by confidence, importance, and durability.

Auto-Consolidation

Every hour Mercury synthesizes a profile summary and generates reflections from patterns it detects. Active memories promote to durable when reinforced 3+ times.

You Stay in Control

/memory gives you overview, search, pause, and clear. Second Brain can be disabled entirely via config. All data stays on your machine in SQLite — nothing leaves your machine.

How It Works

1
Extract

After every conversation, a dedicated extraction pass pulls 0–3 facts. Each fact gets a type, confidence score (0–1), importance, and durability rating. Below 0.55 confidence? Rejected.

2
Store & Merge

Facts land in SQLite with FTS5 full-text search. If a similar fact already exists, Mercury merges — incrementing evidence count, updating confidence, and resolving contradictions. No duplicates.

3
Recall

Before each message, a FTS5 query retrieves the top 5 relevant memories within a 900-character budget. Only what matters enters context. Your token spend stays low.

4
Consolidate & Prune

Every 60 minutes: profile synthesis, reflection generation, and active→durable promotion (3+ evidence). Auto-pruning dismisses stale active memories (21 days) and decays low-confidence durables (120 days).

Why it matters: This isn't a chat history dump. It's a structured knowledge graph that improves with every conversation — the more you use Mercury, the better it gets. And because it uses SQLite + FTS5 on your local machine, recall is instant, search is sub-millisecond, and your data never leaves ~/.mercury/. Enable/disable: SECOND_BRAIN_ENABLED=false env var or memory.secondBrain.enabled: false in ~/.mercury/mercury.yaml. Enabled by default.

Up and Running in 60 Seconds

1

Install

npm i -g @cosmicstack/mercury-agent

Or use npx @cosmicstack/mercury-agent — no install needed.

2

Setup

mercury

First run triggers the onboarding wizard. Choose one or more providers, validate each key by fetching models, pick your default model, and optionally pair Telegram with a bot token plus pairing code.

3

Run

mercury start

Mercury wakes up, loads your soul files, restores scheduled tasks, and runs as a background daemon. Start talking via Telegram or use mercury start --foreground for attached mode.
